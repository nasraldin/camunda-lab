# BPMN semantic diff (`camunda diff`) — Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development or executing-plans. Checkbox tracking. Commits only when asked.

**Goal:** `camunda diff` shows human-readable semantic changes between BPMN (and later DMN/forms) versions — not raw XML git noise.

**Architecture:** Shared `internal/bpmn` parses XML → normalized IR. `internal/diff` compares two IRs and formats added/removed/changed findings. CLI resolves “current file vs git base” or two explicit paths.

**Tech Stack:** Go, encoding/xml, optional `go-git` or `git show` exec for base blobs, Cobra.

**Depends on:** Prefer implementing `internal/bpmn` IR once for lint/diff/explain (see also lint plan). If IR does not exist yet, create it in Task 1 here or land lint IR first.

**Vision:** [../specs/2026-07-17-platform-toolkit-vision.md](../specs/2026-07-17-platform-toolkit-vision.md)

## Global Constraints

- Exit codes: `0` no semantic change, `1` changes found, `2` error
- Output is categorical (Added Service Task, Timer PT5M→PT10M), not unified XML hunks
- MVP BPMN elements only (see below); DMN decision-table row/cell diff is v1.1 follow-on in this plan’s “Later”
- No network; no cluster calls
- Fixtures under `testdata/bpmn/`; no paid APIs in tests
- Does not deploy or mutate cluster state

## BPMN IR (shared)

```go
// internal/bpmn — conceptual shape
type Model struct {
    ProcessID string
    Elements  []Element   // id, type, name, attrs
    Flows     []Flow      // id, source, target, condition
    Messages  []Message
    // Timers, boundary events, retry/timeout extensions as structured fields
}
```

**MVP element coverage:** service / user / script tasks; exclusive/parallel/inclusive gateways + conditions; timer events; error/message boundary events; Camunda 8 retry/timeout extension attrs commonly set on service tasks; message names.
